## Digital ID Module

### Viewing Your Digital ID

1. Tap the **ID Card** tab
2. Your digital ID displays with photo, name, and role
3. Tap the card to flip and see the barcode

### Biometric Protection (Optional)

Protect your ID with Face ID or fingerprint:

1. Go to **Profile** ‚Üí **Settings**
2. Enable "Biometric Protection"
3. Confirm with your device biometric

### Scanning Student IDs (Staff Only)

1. From Home, tap **"Scan ID"** or navigate to ID Scanner
2. Point camera at student's QR code
3. View verification result:
   - ‚úÖ Green = Valid ID
   - ‚ùå Red = Invalid/Expired ID
4. Scan logs are automatically recorded

### Managing ID Photos (Staff/Admin)

1. Go to **Profile** ‚Üí **ID Management**
2. View pending photo submissions
3. Tap a photo to review
4. Select **Approve** or **Reject**
5. Rejected photos require resubmission

## Smart Pass System

### Creating a Pass (For Students)

1. Tap the **Smart Pass** tab
2. Tap **"Request Pass"**
3. Select destination (Bathroom, Nurse, Office, etc.)
4. Add optional note
5. Submit and wait for approval

### Approving Passes (Staff)

1. Go to **Smart Pass** tab
2. View **"Pending Requests"** section
3. Tap a request to review details
4. Select **Approve** or **Deny**
5. Approved passes start immediately

### Hall Monitor View (Staff)

1. From Smart Pass, tap **"Hall Monitor"**
2. View all active passes in real-time:
   - Student name and photo
   - Origin and destination
   - Time elapsed
   - Status indicator (on-time/overtime)
3. Tap any pass for details or to end it

### Pass Features

| Feature | Description |
|---------|-------------|
| **Time Limits** | Passes have automatic time limits |
| **Overtime Alerts** | Visual warning when time exceeded |
| **Capacity Limits** | Destinations have maximum occupancy |
| **No-Fly Times** | Certain times may restrict passes |
| **Encounter Prevention** | System prevents specific student meetups |

## Emergency Communications

### Understanding Alert Types

| Alert Type | Meaning | Action |
|------------|---------|--------|
| **Lockdown** | Active threat | Lock doors, hide, stay silent |
| **Fire/Evacuation** | Building evacuation | Exit via nearest route to assembly point |
| **Shelter in Place** | External threat | Move to interior room, stay inside |
| **Hold** | Hallway incident | Stay in current location |
| **All Clear** | Emergency ended | Resume normal activities |

### Receiving Emergency Alerts

1. Alerts appear as full-screen notifications
2. **DO NOT DISMISS** until you've read instructions
3. Follow the specific guidance displayed
4. Check in when prompted

### Emergency Check-In

When prompted during an emergency:

1. Alert will display check-in options
2. Tap **"I'm Safe"** if you are secure
3. Tap **"Need Help"** if you require assistance
4. Your location and status are reported to administrators

### Triggering Emergency Alerts (Admin Only)

1. Go to **Home** ‚Üí **"Emergency Alert"**
2. Select alert type from templates:
   - Lockdown
   - Fire Evacuation
   - Tornado Warning
   - Medical Emergency
   - And more...
3. Choose severity level
4. Review students currently in hallways
5. Confirm to broadcast alert

### Drill Mode

For practice emergencies:

1. Select "Drill Mode" when triggering
2. Alert will clearly indicate "THIS IS A DRILL"
3. All features work normally for practice
4. Use to train students and staff

## Notifications

### Viewing Notifications

1. Tap the **Messages** tab
2. View all received notifications
3. Unread items are highlighted
4. Tap to view full details

### Sending Notifications (Staff)

1. Go to **Messages** ‚Üí **"Compose"**
2. Enter notification title
3. Enter message body
4. Select target audience:
   - All Users
   - Students Only
   - Parents Only
   - Staff Only
   - Multiple selections
5. Tap **Send**

### Scheduling Notifications (Staff)

1. In Compose screen, tap **"Schedule"**
2. Select date and time for delivery
3. Complete the notification
4. Tap **"Schedule"** to confirm
5. View scheduled items in "Scheduled" tab

### Using Templates

1. In Compose screen, tap **"Templates"**
2. Select from available templates:
   - Early Dismissal
   - School Event Reminder
   - Weather Update
   - General Announcement
3. Customize as needed
4. Send or schedule

### Managing Scheduled Notifications

1. Go to **Messages** ‚Üí **"Scheduled"** tab
2. View all pending notifications
3. Tap to view details
4. Option to cancel if needed

## Visitor Management

### Pre-Registering a Visitor

1. Go to **Profile** ‚Üí **"Visitor Management"**
2. Tap **"Pre-Register Visitor"**
3. Enter visitor details:
   - Full name
   - Email (optional)
   - Purpose of visit
   - Expected date and time
4. Submit registration
5. Share confirmation with visitor

### Visitor Check-In Process

When visitors arrive:

1. Direct them to kiosk or reception
2. Visitor provides name or scans ID
3. System verifies against pre-registration
4. Photo captured for badge
5. Digital badge generated with QR code
6. Host (you) receives arrival notification

### Receiving Visitor Notifications

1. When your visitor checks in, you receive a push notification
2. Notification includes:
   - Visitor name
   - Check-in time
   - Location
3. Proceed to reception to meet your guest

### Checking Out Visitors

1. Visitors must check out when leaving
2. Badge QR code scanned at exit
3. Visit duration recorded
4. Host notified of departure

### Watchlist Alerts

If a visitor triggers a watchlist alert:

1. System immediately notifies security
2. Front desk receives warning
3. Follow school security protocols
4. Do not confront - contact security

## Troubleshooting

### Login Issues

| Problem | Solution |
|---------|----------|
| Forgot password | Tap "Forgot Password" on login screen |
| Account locked | Wait 15 minutes, then try again |
| Invalid credentials | Verify email spelling, reset password if needed |

### App Issues

| Problem | Solution |
|---------|----------|
| App crashes | Force close and reopen |
| Not receiving notifications | Check device notification settings |
| Camera not working | Grant camera permission in device settings |
| Biometric not working | Re-enable in app settings |

### Smart Pass Issues

| Problem | Solution |
|---------|----------|
| Can't create pass | Check if no-fly time is active |
| Pass denied | View reason, try different destination |
| Timer not showing | Refresh the screen |
